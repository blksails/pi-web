# Implementation Plan

## 任务说明
核心实现集中于单一新元件 `PiInteraction`（`elements/pi-interaction.tsx`），任务 1–4 因共享同一文件而**顺序执行、不可并行**；集成（任务 5）与验证（任务 6）涉及不同文件，标注 `(P)` 的子任务可并行。

- [x] 1. 内联交互组件骨架与状态机（Foundation）
- [x] 1.1 搭建 PiInteraction 组件骨架与 FIFO/留痕状态机
  - 定义命令式 props（消费 `useExtensionUI` 结果与协议交互类型，仅类型消费、不改协议）
  - 实现交互类守卫：仅 confirm/select/input/editor 进入内联呈现，ambient method 不在此处理
  - 维护本地已应答留痕状态（组件 `useState`，不写持久层/消息历史）
  - 实现 active 选取：仅 `queue[0]` 且其 id 不在留痕中为可应答；其后排队项不可应答
  - 实现 respond 包装：成功后将该项追加为留痕、失败不追加（供后续错误任务消费）
  - 容器渲染 `data-pi-interaction` + 分组语义；无 extensionUI 能力或无 active 且无留痕时不渲染（降级不报错）
  - 在 elements 入口导出该组件，供装配层引用
  - 完成态：组件可被导入；给定含交互类 `current` 的 mock，渲染出带 `data-pi-interaction` 的容器；无 extensionUI 时渲染为空
  - _Requirements: 2.7, 4.1, 4.2, 4.3, 6.1, 6.3, 8.1, 8.5_

- [x] 2. 四类交互的 active 呈现与应答回传（Core）
- [x] 2.1 confirm 内联卡与批准/拒绝回传
  - 渲染 confirm 请求的标题/描述与「批准」「拒绝」动作
  - 点击批准/拒绝分别以确认通过/否决语义携带请求 id 回传
  - 完成态：点击批准触发以确认通过语义的回传，点击拒绝触发以否决语义的回传，二者均带匹配 id
  - _Requirements: 1.1, 1.3, 2.1, 2.2_
  - _Boundary: PiInteraction_

- [x] 2.2 select/input/editor 内联卡与 value 提交及取消回传
  - select：渲染选项分组并以所选值提交回传
  - input：渲染单行输入并以输入文本提交回传
  - editor：渲染多行编辑（含可选预填文本）并以编辑后文本提交回传
  - 三类均提供「取消」以取消语义回传
  - 完成态：select 提交回传所选值、input/editor 提交回传输入文本、取消回传取消语义，均带匹配 id
  - _Requirements: 1.1, 1.3, 1.4, 2.3, 2.4, 2.5, 2.6_
  - _Boundary: PiInteraction_

- [x] 3. 应答终态留痕呈现（Core）
- [x] 3.1 已应答只读终态卡渲染
  - 成功应答后保留该卡为只读终态而非移除
  - 终态文案：confirm→已批准/已拒绝；select→已选择所选值；input→已提交输入值；editor→已提交（过长文本折叠或省略）；取消→已取消
  - 留痕按应答先后排序，active 卡位于既有留痕之后（最新）
  - 终态卡不再接受进一步输入或再次提交
  - 完成态：依次应答多个请求后，界面保留对应只读终态卡且文案与各自结果一致，最新 active 排在留痕之后
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 4.4_
  - _Boundary: PiInteraction_

- [x] 4. 可达性与错误重试（Core）
- [x] 4.1 active 卡可达性行为
  - 新 active 请求出现时以非打断优先级实时播报、滚动至可见、聚焦其首个可操作控件
  - 不锁定焦点，允许用户离开该卡继续在输入框输入
  - 为交互卡提供分组语义与可访问名称
  - 完成态：新 active 出现时其首动作控件获得焦点且卡可见，焦点未被锁定（可 tab 离开）
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5_
  - _Boundary: PiInteraction_

- [x] 4.2 应答失败保留、错误展示与重试
  - 应答提交失败时保留该卡为可应答状态并展示可读错误信息
  - 失败后再次提交可重新尝试回传
  - 提交进行中禁用该卡的提交与动作控件以防重复提交
  - 完成态：模拟回传失败后该 active 卡仍在并显示错误，重试可再次发起回传；提交中动作控件为禁用态
  - _Requirements: 7.1, 7.2, 7.3, 7.4_
  - _Boundary: PiInteraction_
