# Implementation Plan

- [x] 1. Foundation: 协议 DTO
- [x] 1.1 新增会话操作与会话收藏的传输契约
  - 在 protocol 传输契约中新增删除请求、重命名请求/响应、会话收藏列表响应、会话收藏设置请求的 schema 与推断类型
  - 校验规则:sessionId 非空;重命名 name trim 后非空且限长(≤200);收藏集合为字符串数组(去重、丢空串)
  - 观察完成:新 schema 可对合法负载解析通过、对空 sessionId / 空名 / 超长名 / 非数组收藏拒绝;对应类型可被 server 与 react 引用
  - _Requirements: 2.3, 3.2, 4.1, 4.2, 6.4_
  - _Boundary: Protocol rest-dto_

- [x] 2. 会话操作服务端(存储写接缝 + 端点)
- [x] 2.1 (P) 实现会话收藏偏好存储
  - 以原子写 JSON 范式(临时文件 + rename)读写 `<agentDir>` 下的会话收藏文件,内容为收藏的 sessionId 集合
  - 容错:文件缺失 / 坏 JSON 返回空集不报错;写入去重、丢空串;读回落盘结果保持一致
  - 观察完成:单测证明缺失/坏文件 → 空集、set 后 list 往返一致且去重、并发写不产生半写
  - _Requirements: 4.1, 4.2, 4.6, 4.8_
  - _Boundary: SessionFavoritesStore_
  - _Depends: 1.1_

- [x] 2.2 实现会话操作路由组(删除 / 重命名 / 收藏)
  - 新增无 `:id` 路径参数、统一 POST(读收藏用 GET)的路由组,sessionId 走请求体/查询,绕过内存会话存在性门控,挂在既有会话 API 转发器下
  - 删除:调用会话事件存储物理删除;目标不存在按幂等成功返回;重命名:向目标会话追加一条会话信息条目(新生成 id、parentId 置空、当前时间戳)使其成为最新显示名,目标不存在返回未找到
  - 收藏:读返回当前收藏集合、写全量替换并回显落盘结果;写操作门控关闭时三个写端点拒绝且不触达存储,收藏读不受门控
  - body 经校验非法返回校验错误;从 server 包对外导出该路由组构造器
  - 观察完成:单测覆盖删除已存在/不存在(幂等)、重命名成功/空名超长(400)/不存在会话(404)、收藏读写往返、门控关闭时写 403 且读 200
  - _Requirements: 2.3, 2.6, 2.8, 3.2, 3.7, 4.1, 4.2, 4.9, 6.2, 6.3, 6.4_
  - _Boundary: SessionActionsRoutes_
  - _Depends: 1.1, 2.1_

- [x] 2.3 会话操作跨后端持久化集成测试
  - 针对真实文件与 sqlite 两种后端,验证重命名追加后可完整回放历史且显示名反映新名、删除后列表与头部读取不再返回该会话
  - 观察完成:两种后端下集成测试均绿,证明重命名与删除的持久化效果跨后端一致
  - _Requirements: 2.3, 3.2, 3.3_
  - _Depends: 2.2_
