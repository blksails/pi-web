# Implementation Plan

- [ ] 1. 基础：回归基线与共享契约原语
- [x] 1.1 采集本期改动前的四面回归基线
  - 分别运行递归子包单测、根应用测试、离线 Node 端到端、递归类型检查四个测试面，各自记录通过/失败计数
  - 运行既有浏览器 e2e 检阅并逐项记录结果
  - 观察态：五份基线快照落到 spec 目录下可复查的记录中，任何**改动前即失败**的用例被显式标注为既有问题
  - 说明：四面必须分别采集——后续会改动包主入口，而主入口是根应用与其它子包的消费面，只跑单个包会漏红
  - _Requirements: 10.2, 10.3_

- [x] 1.2 契约版本常量与装配期版本自检
  - 提供整个契约共用的版本常量与不一致即抛的自检函数
  - 版本判定为严格相等，不提供向下兼容分支
  - 错误携带双方版本号
  - 观察态：以不一致版本调用自检会抛出携带期望值与实际值的类型化错误；一致时静默返回
  - _Requirements: 9.1, 9.2, 9.3_

- [x] 1.3 宿主状态存储的类型契约与错误分类
  - 定义双根工作区与命名空间的方法签名、写入选项、JSON 对象类型
  - 定义四类错误及其稳定判别码，错误携带键名等上下文字段
  - 端口对象自带版本字段，使实现在类型层无法声明错误版本
  - **只交付类型与错误，不建该模块的出口文件**——出口由 6.1 在全部实现就位后统一收口，避免出口引用尚不存在的符号而使整段核心期的类型检查为红
  - 观察态：四类错误均可经判别码区分且各自携带上下文；类型可被其它模块按具体文件路径 import 而无需任何实现
  - _Requirements: 2.1, 2.2, 3.4, 4.1, 9.1_

- [ ] 2. 核心：纯逻辑原语
- [x] 2.1 (P) 键空间校验
  - 拒绝相对段、绝对路径、空段、尾随分隔符、反斜杠、空字符与空串
  - 接受以分隔符连接的多段相对键；键按大小写敏感处理
  - 校验在触及任何存储之前完成
  - 观察态：穷举非法形态的单测全绿，每例断言判别码为键错误且错误携带原键
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6_
  - _Boundary: workspace/key，不建也不改模块出口_

- [x] 2.2 (P) 单键值上限的环境变量解析
  - 未设置时采用默认上限；设为正整数时以该值为准
  - 不可解析、非整数、非正值一律在装配期抛错，不静默回落默认
  - 解析函数接受注入的环境变量对象而非直接读取全局
  - 观察态：按「缺省/合法/各类非法」分组的单测全绿，非法各形态逐一断言抛错
  - _Requirements: 3.1, 3.2, 3.3_
  - _Boundary: workspace/limit-config，不建也不改模块出口_

- [x] 2.3 (P) 深度合并语义
  - 嵌套对象递归合并、数组整体替换、忽略未定义值
  - 覆盖模式使既有值中本次未提供的字段被删除
  - 观察态：单测断言合并结果与既有配置落盘器的行为逐项一致，含数组替换与删除语义
  - _Requirements: 2.3, 2.4_
  - _Boundary: workspace/merge，不建也不改模块出口_

- [ ] 3. 核心：一致性测试套件
- [x] 3.1 套件骨架、测试子模块出口与键空间用例组
  - 定义最小测试框架契约、验收目标类型与**可携带上限的工厂参数类型**，套件本身不 import 任何测试框架，断言使用内置断言库
  - 错误判定一律按稳定判别码，不使用构造函数判定
  - 交付测试子模块的出口文件，作为对外子路径的入口
  - 覆盖键空间全部非法与合法形态
  - 交付两个**测试夹具**内存实现（一个合规、一个故意不校验键），落测试镜像目录，**不经对外子路径导出**
  - 观察态：套件可被传入任意 describe/it 实现驱动；对违规夹具运行时键空间用例组失败，对合规夹具运行时通过
  - _Requirements: 8.1, 8.2, 8.6, 1.1, 1.2, 1.3, 1.4, 1.5, 1.6_
  - _Depends: 1.3_

- [x] 3.2 读写语义与双根隔离用例组
  - 覆盖缺失键返回空、损坏值抛错、两种合并模式、非递归且字典序的列举、幂等删除、存在性查询、读己之写
  - 覆盖同键分写两根互不覆盖、单根删除不影响另一根、单根列举不串另一根
  - 观察态：用例组逐条锚定验收编号，且对 3.1 的合规夹具全部通过
  - _Requirements: 8.2, 2.1, 2.2, 2.3, 2.4, 2.5, 2.7, 2.8, 2.9, 4.1, 4.2, 4.3, 4.4_

- [x] 3.3 上限三例与并发原子可见性用例组
  - 上限三例：写超限抛错、经工厂参数指定更大上限后新上限生效、上限调小后既有超限值仍可读
  - 上限用例经工厂参数取得不同上限的实例，**不得改写进程环境**
  - 并发用例以并发编排发起多次互不相同的整值写入与若干次读取，断言每次读回完整等于某一次写入或初始空值，不接受字段混合体
  - 并发用例不使用固定等待
  - 观察态：三例上限用例与并发用例均在套件内，且工厂被以不同上限多次调用并产出相互隔离的实例
  - _Requirements: 8.3, 8.4, 8.6, 2.6, 3.4, 3.5_

- [ ] 4. 核心：本地参照实现
- [ ] 4.1 本地实现的键映射、基本读写与合并落地
  - 键映射为根目录下的相对路径，映射使用路径拼接而非字符串连接
  - 缺失键读为空对象；内容非法时抛损坏错误；其余读写失败抛 IO 错误
  - 写入按两种合并模式落地既有合并语义
  - 列举只返回直接子级中持有值的键并按字典序升序
  - 观察态：针对本地实现的直接单测覆盖读缺失、读损坏、两种合并模式、列举非递归与幂等删除
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.7, 2.8, 2.9_
  - _Depends: 2.1, 2.3_

- [ ] 4.2 原子写入、落盘权限与写时上限校验
  - 写入经同目录临时文件后重命名，使读取方绝不观察到部分写入
  - 目录与文件权限与既有配置落盘器一致
  - 上限只在写入路径校验，读取路径不设限
  - 观察态：以大于当前上限的既有值验证读取仍完整返回，同时写入被拒并抛超限错误
  - _Requirements: 2.6, 3.4, 3.5_
  - _Depends: 2.2_

- [ ] 4.3 双根装配与上限接线
  - 用户级根与项目级根分别装配，两者相互隔离
  - 上限缺省取自环境变量解析结果，可由构造参数覆盖
  - 观察态：以两个不同临时目录构造实例后，同键分写两根各自独立读回
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 3.1_

- [ ] 4.4 令本地实现通过完整一致性套件
  - 在既有测试目录中以本地实现为目标驱动套件
  - 工厂接受上限参数并每次调用产出使用独立临时根的隔离目标，用例间不串数据
  - 观察态：套件全部用例对本地实现通过，且用例数与套件声明的用例组一致
  - _Requirements: 8.5, 8.6_
  - _Depends: 3.3, 4.3_

- [ ] 5. 核心：其余三端口
- [ ] 5.1 (P) 能力授予契约类型

  - 附件授予仅在带会话标识请求时出现
  - 结构化定义出口授予类型，不 import 引入 pi SDK 值的模块
  - 本模块自建其出口文件（单任务独占，无并行争抢）
  - 观察态：类型层结构兼容断言通过，证明出口授予类型与既有出口输入形状可互换
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5, 5.6, 5.7_
  - _Boundary: capability_

- [ ] 5.2 (P) 能力面名册与组装引擎
  - 冻结 v1 的能力标识名册，**恰 16 项**：15 个路由能力 + 1 个非路由的宿主命令能力。既有 17 个注入式路由工厂中，两个工具领域的模型清单端点不入名册（按设计判定为领域泄漏、将被删除而非表态），故 17 − 2 = 15
  - 组装时先校验未知标识、后校验缺失表态，两者均抛携带标识列表的错误
  - 弃用表态要求非空原因并经回调通知；沿用与替换分别采用默认与宿主工厂
  - 路由类型与依赖对象均泛型化，不依赖既有 HTTP 模块
  - 观察态：单测覆盖缺表态、未知标识、空白原因、替换生效与弃用回调五种情形，并断言名册长度恰为 16
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5, 6.6, 6.7_
  - _Boundary: host-manifest_

- [ ] 5.3 (P) 配置域注册表与宿主默认域

  - 观察态：单测断言默认注册集恰为四个宿主域且不含工具领域域
