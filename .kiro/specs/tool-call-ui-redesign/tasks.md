- [x] 1. 定制契约与渲染入口扩展（Foundation）

- [x] 2. 工具卡复合化（Core）
- [x] 2.1 拆出工具卡头部子组件
  - 头部子件渲染工具名、状态徽章与折叠触发器；徽章覆盖四态（Running / Streaming 带旋转图标 / Completed / Error），错误态使用 destructive 配色
  - 保留状态徽章容器的数据属性；折叠触发器提供展开态与受控目标的无障碍属性且键盘可达
  - 观察完成：四种状态分别渲染对应徽章文案，流式态徽章带旋转图标，触发器可聚焦可激活
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 7.1, 7.3_
  - _Boundary: ToolHeader_
- [x] 2.2 拆出明细容器子组件
  - 明细容器按展开状态控制可见性，承载明细区数据属性与被无障碍属性引用的标识，错误态文字使用 destructive 配色
  - 观察完成：折叠时不渲染明细、展开时渲染明细，且其标识与头部触发器的受控目标一致
  - _Requirements: 3.4, 7.2_
  - _Boundary: ToolContent_
  - _Depends: 2.1_
- [x] 2.3 实现入参语法高亮子组件
  - 入参子件将工具入参序列化为 JSON 代码块并经既有富文本渲染原语呈现语法高亮，不引入新依赖；序列化失败时回退为纯文本，入参为空时不渲染
  - 观察完成：处于输入态的工具卡以语法高亮代码块呈现入参，且未新增第三方依赖
  - _Requirements: 4.1_
  - _Boundary: ToolInput_
  - _Depends: 2.2_
- [x] 2.4 实现输出富渲染子组件
  - 输出子件接受任意富节点作为输出并在缺省时以富文本渲染原语呈现；存在错误文本时渲染错误文本并使用 destructive 配色；非字符串输出安全序列化、序列化失败回退字符串
  - 观察完成：错误态显示红色错误文本，对象型输出以 JSON 形式呈现，可注入自定义富节点
  - _Requirements: 4.2, 4.3, 4.4_
  - _Boundary: ToolOutput_
  - _Depends: 2.3_
- [x] 2.5 重写装配壳并落地按状态展开策略
  - 默认工具卡改为由头部、明细容器、入参、输出四子件装配；保留既有的状态与工具名推导；未显式指定展开时，完成态与错误态默认展开、输入态与流式态默认折叠，显式展开值优先；根容器保留全部根级数据属性
  - 观察完成：完成/错误态默认展开、输入/流式态默认折叠、显式展开值覆盖默认；根容器数据属性（标记、阶段、工具名）齐全且阶段取值仍限定四态
  - _Requirements: 1.2, 1.4, 3.1, 3.2, 3.3, 7.1_
  - _Boundary: PiToolPart_
  - _Depends: 2.1, 2.2, 2.3, 2.4_
