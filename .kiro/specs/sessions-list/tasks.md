# Implementation Plan

> 前置：worktree 依赖已安装。所有改动落在 worktree。改注入路由/配置域后需重启 dev（端口 3010）。e2e 用隔离 build（`NEXT_DIST_DIR=.next-e2e` + external server）。

- [x] 1. 基础：共享列表契约
- [x] 1.1 定义会话列表的请求/响应/列表项数据契约
  - 在协议层新增「列表请求」（视图范围、目录、单页上限、续取游标）、「列表项」（标识、名称、所属目录、创建时间、可选最近更新时间）、「列表响应」（列表项数组、可选下一页游标、生效视图范围、系统视图是否启用）三组契约。
  - 与现有协议同款方式（schema + 推导类型）定义并导出，供前后端共用。
  - 观察完成：协议包构建通过，新增契约可被服务端与前端导入；对畸形负载能被校验拒绝。
  - _Requirements: 2.4, 3.1, 3.3_
  - _Boundary: ListSessions DTO_

- [x] 2. 核心实现
- [x] 2.1 (P) 实现只读会话列表端点
  - 提供 `GET /sessions`：按视图范围读取「当前目录」或「系统（全机器）」会话；仅使用轻量头部元数据，不读取会话正文。
  - 排序采用「最近更新时间回退到创建时间」的倒序，保证跨存储后端一致。
  - 内存切片分页：依单页上限与续取游标返回一页，并给出下一页游标；续取不重复已返回会话。
  - 系统视图未启用时拒绝系统范围请求并返回明确的「未启用」结果，且不触达存储。
  - 单个会话元数据损坏时跳过该会话、继续返回其余会话，不使整个请求失败。
  - 端点经现有外部路由注入接缝产出（一组可注入路由），不修改内置路由表。
  - 观察完成：对某目录请求返回该目录会话且最新在前；带游标续取得到不重复的后续页；系统视图关闭时系统范围请求返回未启用、开启时返回聚合分页结果。
  - _Requirements: 1.1, 1.2, 1.4, 2.1, 2.3, 2.4, 3.2, 3.3, 3.4_
  - _Boundary: createSessionListRoutes_

- [x] 2.2 (P) 实现前端列表调用封装
  - 在前端客户端新增「列出会话」方法：按视图范围/目录/分页参数请求列表端点并返回类型化结果。
  - 观察完成：调用该方法可拿到类型化的列表响应；空结果返回空数组而非抛错。
  - _Requirements: 6.1_
  - _Boundary: PiClient.listSessions_
  - _Depends: 1.1_

- [x] 2.3 (P) 实现会话列表面板组件
  - 展示会话项：名称或标识、时间、所属工作目录；提供「恢复」操作回调。
  - 视图切换：在「当前目录」与「系统会话」间切换并指示当前视图；系统会话入口仅在系统视图启用时出现。
  - 状态可见：加载中显示加载态；无会话显示空态；加载失败显示可重试错误态。
  - 分页：提供「加载更多」依下一页游标续取。
  - 仅展示头部元数据，不请求会话正文。
  - 观察完成：给定列表数据函数与启用标志，组件渲染列表项与（按启用标志的）切换控件，并在空/加载/错误三态下呈现相应提示；点击「恢复」触发回调并带会话标识。
  - _Requirements: 1.3, 2.2, 3.1, 3.2, 6.1, 6.2, 6.3_
  - _Boundary: SessionListPanel_
  - _Depends: 1.1_

- [x] 3. 集成接线
- [x] 3.1 装配后端列表端点与系统视图门控
  - 在处理器装配处注入会话列表路由，复用与冷恢复相同的存储配置来源（同一后端）。
  - 由部署配置决定系统视图是否启用，并将该门控传入端点；默认关闭。
  - 观察完成：应用运行后 `GET /sessions` 在 `/api` 下可达并返回当前目录会话；未设开关时系统范围请求返回未启用。
  - _Requirements: 2.1, 2.3_
  - _Depends: 2.1_
  - _Boundary: pi-handler 接线_

- [x] 3.2 宿主接线：注入展示位置、全局开关与恢复
  - 在聊天宿主读取「系统视图开关」与「展示位置」两项部署配置：默认关闭系统视图、默认展示于左侧栏区域，可经配置重定位到其它宿主区域。
  - 经宿主插槽优先级链注入列表面板，与既有/扩展内容追加共存，不替换或遮挡对话区。
  - 「恢复」回调复用既有按会话标识恢复链路进入会话并回放历史；恢复失败时提示且不丢失当前会话。
  - 系统视图关闭时不渲染系统会话入口。
  - 观察完成：默认配置下左侧栏出现会话列表且无系统会话入口；点击某会话「恢复」进入该会话并显示其历史消息；改展示位置配置后面板出现在对应区域。
  - _Requirements: 2.2, 4.1, 4.2, 4.3, 5.1, 5.2, 5.3, 5.4_
  - _Depends: 2.2, 2.3, 3.1_
  - _Boundary: chat-app 接线_

- [x] 4. 验证
- [x] 4.1 端点单元/集成测试
  - 覆盖：当前目录视图按时间倒序返回；分页游标续取不重复并收敛；系统视图关闭时系统范围请求被拒绝且不读存储；游标解码失败返回参数错误；列表响应契约对畸形负载拒绝。
  - 经真实文件存储后端集成验证当前目录与系统范围（启用时）聚合。
  - 观察完成：测试套件通过，上述断言全部成立。
  - _Requirements: 1.1, 1.2, 1.4, 2.1, 2.3, 2.4, 3.2, 3.3, 3.4_
  - _Depends: 2.1, 3.1_
  - _Boundary: createSessionListRoutes_

- [x] 4.2 浏览器 e2e（隔离 build）
  - 在隔离构建 + external server 模式下验证关键用户路径：
    - 列当前目录会话与空态可见性。
    - 系统视图开关 on/off：关→无系统会话入口；开→可切换并分页。
    - 展示位置重定位：默认左侧栏 vs 另一区域，面板就位且不遮挡对话区。
    - 从列表点击「恢复」进入会话并回放历史消息。
  - 观察完成：e2e 全部用例通过，截图/断言佐证四条路径成立。
  - _Requirements: 1.1, 1.3, 2.2, 2.4, 4.1, 4.2, 5.1, 5.2, 5.3, 6.1_
  - _Depends: 3.2_
