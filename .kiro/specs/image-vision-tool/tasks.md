# Implementation Plan

- [x] 1. 基础：契约层与失败枚举
- [x] 1.1 建立视觉识别的结果契约与失败原因枚举
  - 定义成功结果（文字结论 + 实际所用模型标识）与失败结果（原因 + 可选说明）的判别联合
  - 定义十种互斥失败原因：附件能力不可用、会话无图、引用无法解析、附件非图像、无可用视觉模型、指定模型不可用、用户取消、已中止、凭据解析失败、模型调用失败
  - 定义调用方入参形状（图像引用可选、问题必填、模型可选）与已取回图像的形状（裸 base64 + mimeType + 来源 id）
  - 定义依赖注入契约，使内核的模型调用与附件上下文获取可在测试中替换
  - 提供失败结果构造器；约束失败结果永不携带图像字节
  - 完成态：类型通过 `pnpm typecheck`，且存在一组断言证明十种失败原因两两可区分
  - _Requirements: 7.2_

- [x] 2. 核心：图像来源解析与模型选择
- [x] 2.1 (P) 实现图像来源解析
  - 按附件引用取回图像字节，转为**裸 base64**（不带 `data:` 前缀）
  - 未提供引用时，枚举当前会话附件，筛出图像类型，按创建时间降序取最新一张
  - 引用无法解析时归为「引用无法解析」；附件非图像类型时归为「附件非图像」；会话内无任何图像时归为「会话无图」
  - 附件上下文解析过程中抛出的任何异常都被捕获并映射为失败结果，不外泄
  - 不复用既有的 data URI 取图工具（其产出带前缀），在本模块内自持转换
  - 完成态：单元测试覆盖多图选最新、跳过非图像附件、三类失败分支，且断言产出无 `data:` 前缀
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_
  - _Boundary: resolve-image_

- [x] 2.2 (P) 实现视觉模型候选、交互选择与降级链
  - 候选清单 = 模型注册表中「凭据可用」且「支持图像输入」的模型；不得使用返回全部模型的接口
  - 不维护任何静态模型清单，使新增配置的图像输入模型自动成为候选
  - 有交互界面且调用未指定模型时，提示用户从候选中选择；选择器返回的是标签字符串，需维护标签到模型的反查
  - 用户取消选择时归为「用户取消」；调用显式指定但不在候选中的模型归为「指定模型不可用」
  - 无交互界面时绝不触发选择器，按序降级：显式模型 → 环境变量默认模型 → 候选首个 → 「无可用视觉模型」
  - 模型标识统一为 `provider/modelId` 形式
  - 完成态：单元测试覆盖降级链四个分支与两类失败；并断言无交互界面时选择器调用次数为零
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 3.4, 4.1, 4.2, 4.3, 4.4, 4.5_
  - _Boundary: select-model_

- [x] 3. 编排：识别内核
- [x] 3.1 实现识别内核并显式解析模型凭据
  - 编排顺序：附件能力可用性检查 → 取图 → 选模型 → 解析凭据 → 调用模型 → 组装结果
  - **显式解析凭据**：从模型注册表取得该模型的 apiKey / headers / env 并传入模型调用；不得依赖模型调用方自行回落环境变量（目标 provider 的凭据只存在于模型配置文件中）
  - 凭据解析失败归为「凭据解析失败」，与「模型调用失败」区分
  - 构造单条用户消息，内容为文本段 + 图像段两部分，图像段的数据为裸 base64
  - 传入中止信号；中止时归为「已中止」
  - 从模型应答中拼接全部文本段作为结论；结论为空时归为「模型调用失败」
  - 整体包裹异常兜底，内核永不抛出；结果只含文字，绝不含图像字节
  - 完成态：单元测试断言传给模型调用的选项确实携带来自注册表的 apiKey/headers/env，且消息内容为「文本 + 图像」两段、图像数据无 `data:` 前缀；另有测试覆盖凭据失败、调用抛错、中止三条路径且均不抛出
  - _Depends: 2.1, 2.2_
  - _Requirements: 5.1, 5.4, 5.5, 5.6, 6.5, 7.1_
  - _Boundary: run-vision-tool_

- [x] 4. 适配层：工具入口与命令入口
- [x] 4.1 (P) 实现 image_vision 工具适配
  - 以 `image_vision` 之名注册工具，声明图像引用、问题、模型三个参数（后两者分别为必填与可选）
  - 执行时把参数与中止信号交给识别内核
  - 结果内容部分只放文字（结论或失败说明），结构化明细部分放完整结果对象
  - 结果内容中不得出现内联图像，避免被既有的 base64 剥离闸门处理
  - 完成态：集成测试断言工具注册名正确，且成功路径的结果内容仅含文本段、含所用模型标识
  - _Depends: 3.1_
  - _Requirements: 5.2, 5.3, 5.4_
  - _Boundary: tools/image-vision_

- [x] 4.2 (P) 实现 /img_vision 命令适配
  - 以 `img_vision` 之名注册命令；命令处理器无返回值，故结论只能经交互界面通知呈现
  - 整段命令参数作为提问；参数为空时使用默认提问；图像固定走「最近一张图」缺省规则
  - 中止信号取自上下文（命令处理器不接收该形参）
  - 成功以信息级通知呈现结论；调用失败以错误级通知呈现；用户取消与已中止以信息级呈现，不渲染为故障
  - 处理器内部吞掉一切异常，绝不中断会话
  - 完成态：集成测试断言处理器在成功/失败/取消三种路径分别以正确级别调用了通知，且处理器返回空值且不抛出
  - _Depends: 3.1_
  - _Requirements: 6.2, 6.3, 6.4_
