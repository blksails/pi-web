## 0. 被测对象能力概述

`webext-renderer-agent` 是 **Tier2 自定义渲染器**示例,`.pi/web/web.config.tsx` 用
`defineWebExtension` 注册了两类渲染器(`capabilities: ["renderers"]`):

| 注册类型 | key | 渲染组件 | 命中条件 |
|---|---|---|---|
| `renderers.dataParts` | `data-metric` | `MetricRenderer`(`data-testid="metric-card"`) | 消息含 `data-metric` 数据部件 |
| `renderers.tools` | `echo` | `EchoToolRenderer`(`data-testid="echo-tool-card"`,文案「🔧 扩展自定义 echo 工具渲染器(webext-renderer)」) | 消息含 `echo` 工具调用 |

> 该扩展**只注册渲染器、不声明任何区域插槽 / 布局 / 主题**,故首屏不会出现
> `data-pi-ext-*` 插槽 DOM 标记或右侧面板 —— 这是预期,不是"未加载"。

### 关键前提:自定义渲染器的触发条件
渲染器只在**消息流里真的出现对应部件**时才显形:
- `webext-renderer-agent/index.ts` 仅有 `systemPrompt`,**自身不定义任何工具、也不发 `data-metric` 数据部件**。
- 因此**真实 LLM dev 服务**(`localhost:3000`,无 `PI_WEB_STUB_AGENT`)下,与该 source 普通对话**不会**触发自定义渲染器 —— agent 只回纯文本。
- 触发自定义渲染器需 **stub agent**(`PI_WEB_STUB_AGENT=1`):stub **每轮发一个 `echo` 工具**(`tool_execution_start → _end`),命中 `EchoToolRenderer`。这与现存通过的浏览器 e2e `e2e/browser/tool-call-ui.e2e.ts:86-94`(断言 `echo-tool-card` 可见 + 默认 `[data-pi-tool]` 计数为 0)一致。

为兼顾"真实 dev 加载"与"自定义渲染器视觉验证",本次在**两个隔离实例**上分别验收:
1. 共享真实 dev(`localhost:3000`)—— 验证加载 / 渲染 / 会话持久化。
2. **隔离 stub 实例**(`localhost:3011`,`NEXT_DIST_DIR=.next-stub`,独立端口与 dist,从主仓启动,**不触碰 :3000 的 `.next`**)—— 触发并验证自定义 echo 渲染器、暗色对比度、布局。

### 1.1 结构化校验(stub 会话,`evaluate_script`)
```json
{ "echoToolCard": 1, "metricCard": 0, "defaultToolCard": 0,
  "echoText": ["🔧 扩展自定义 echo 工具渲染器(webext-renderer)"] }
```
```json
{ "panelRightPresent": false, "inputDockPresent": true,
  "viewportW": 1271, "horizontalOverflowCount": 0, "scrollW": 1271, "extAttrs": [] }
```
- `echoToolCard=1` 且 `defaultToolCard=0` —— 自定义渲染器经注册表**覆盖**默认工具卡(Req 5.2 / 6.2 / 6.3),UI 控制层生效。
- `metricCard=0` —— 符合预期:stub 发 `echo` 工具而非 `data-metric` 数据部件,故 `MetricRenderer` 未触发(本 source 在任何车道下都不产出 `data-metric`,见 §3)。

### 1.2 暗色对比度(`getComputedStyle`)
```json
{ "theme": "dark", "bodyBg": "rgb(9,9,11)", "bodyColor": "rgb(250,250,250)",
  "echoCardColor": "rgb(250,250,250)", "echoCardBorder": "rgb(124,58,237)" }
```
近白文字(250,250,250)对近黑底(9,9,11),对比度 ~19:1,远超 WCAG AAA。

## 3. 发现与建议(非阻断)

1. **`data-metric` 渲染器在当前任何车道下都无法被触发。**
   `MetricRenderer` 注册了 `data-metric` 数据部件渲染器,但全仓**无任何产出 `data-metric` 部件的代码**(stub agent 也只发 `echo` 工具)。该渲染器目前是"已注册但不可达"。
   - 建议:若要让示例完整自证,可在 stub agent 或该 agent 的行为层补一条 `data-metric` 数据部件的产出路径(类比 echo),或在示例文档注明 `data-metric` 仅供单元/契约级验证。

2. **扩展渲染器内联色值未跟随主题。**
   `EchoToolRenderer` 边框硬编码 `#7c3aed`、`MetricRenderer` 边框硬编码 `#ddd`。echo 紫边在明暗两态都可见、文字继承宿主前景故对比无碍;但 `MetricRenderer` 的 `#ddd` 浅灰边在暗底下会偏弱(本次未触发到 metric 卡,属潜在观察)。
   - 建议:示例渲染器改用宿主 CSS 变量(如 `var(--border)`)以贴合明暗主题。
