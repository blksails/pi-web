# batch-file-check · 批量文件核对

## 何时触发

- 用户给出一份文件清单（聊天粘贴、csv、txt），要核对磁盘上是否齐套
- 批量检查命名是否符合规范（前缀、日期、扩展名）
- 目录巡检：过大文件、空文件、可疑后缀

## 步骤

1. 取得**清单来源**与**根目录**（cwd 或用户指定）。清单若在文件中，用 **`bash`**（`cat`/`head`）读取，**禁止** `read`/`glob` 工具。
2. 解析清单为路径列表（去空行、去注释 `#`）。
3. 对每条路径用 **`bash`** 检查：`存在 / 缺失 / 是目录误当文件`。
4. 可选规则（用户提出才强制）：
   - 扩展名白名单（如仅 `.pdf` `.docx`）
   - 单文件大小上限
   - 命名正则（如 `^\d{8}_.+\.pdf$`）
5. 输出汇总表 + 缺失列表 + 违规列表；缺失数=0 且违规=0 则「通过」。

## 输出模板

```markdown
## 批量核对：通过 | 部分缺失 | 失败

- 清单条数: N
- 存在: A · 缺失: B · 违规: C

### 缺失
1. path

### 违规
1. path — 原因
```

## 纪律

- 大批量时用脚本化 **bash** 一次跑完，避免对每个文件单独 round-trip 上百次。
- 不擅自删除违规文件；只报告，修复需用户确认。
- 本 agent 未启用 `read`/`glob`/`write`/`edit`；文件 IO 一律 bash。
