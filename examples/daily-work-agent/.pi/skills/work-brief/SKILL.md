# work-brief · 工作简报

## 何时触发

- 写日报 / 周报 / 迭代小结
- 把零散聊天或笔记整理成待办
- 会议纪要结构化（决议 / 待办 / 风险）

## 步骤

1. 收集输入：用户粘贴的要点、或指定仓库内 markdown/日志路径（用 **`bash`** 的 `cat`/`find`，禁止 `read`/`glob` 工具）。
2. 归类为：**已完成 / 进行中 / 阻塞 / 下一步**。
3. 每条尽量包含：做什么、为什么（可选）、负责人（若可知）、截止（若可知）。
4. 输出简洁中文简报；若用户要落盘，写入其指定路径（默认不写文件）。

## 输出模板

```markdown
# 工作简报（日期）

## 已完成
- …

## 进行中
- …（状态：…）

## 阻塞 / 风险
- …

## 下一步
- [ ] …
```

## 纪律

- 不编造未提供的完成项或数据。
- 敏感信息（客户名、密钥）按用户要求脱敏。
