# review-file-copy · 审核文件拷贝

## 何时触发

用户要求：

- 审核 / 验收一次文件或目录拷贝
- 核对源路径与目标路径是否一致
- 检查拷贝遗漏、多余、损坏
- 迁移 / 同步后的 diff 报告

## 需要向用户确认的信息

若未给出，先问清再动手：

1. **源路径**（source）
2. **目标路径**（destination）
3. 是否递归子目录（默认是）
4. 是否比对内容哈希（大目录可抽样）
5. 是否允许修改目标（默认只读审核，不写）

## 步骤

1. **存在性**：仅用 **`bash`**（如 `test -d` / `ls`）确认源与目标均存在；任一方缺失则直接失败报告。
2. **清单**：仅用 **`bash`** 列两边相对路径（`find` / `ls -R` 等，优先可复现命令）。**禁止** `read`/`glob`/`write`/`edit` 工具。
3. **集合 diff**：
   - 仅源有 → **遗漏（未拷贝）**
   - 仅目标有 → **多余（目标脏数据或旧文件）**
   - 两边都有 → 进入体量/内容比对
4. **元数据**：对交集文件比对 size（及 mtime 若用户关心）；记录不一致项。
5. **内容抽样**（可选但推荐）：
   - 小文件 / 关键配置：全文或 `shasum`/`md5` 对比
   - 大目录：按扩展名或顶层目录抽样 N 个（默认 5–20）
6. **权限与符号链接**（若相关）：是否跟随 symlink、可执行位是否保留。
7. **输出报告**（固定结构，中文）：

```markdown
## 拷贝审核结论：通过 | 有风险 | 失败

| 项 | 源 | 目标 |
|----|----|------|
| 根路径 | … | … |
| 文件数 | n | m |
| 总大小(约) | … | … |

### 遗漏（源有目标无）
- …

### 多余（目标有源无）
- …

### 内容/大小不一致
- path — 源 size / 目标 size（或 hash）

### 抽样校验
- path — match | mismatch

### 建议
- …
```

## 纪律

- **默认只读**：不自动 `cp`/`rsync` 修复，除非用户明确要求「按审核结果补拷」。
- 补拷前复述将执行的命令与影响范围，得到确认后再 `bash`。
- 路径含空格时正确引用；避免对系统目录（`/etc`、`/`）做宽泛递归。
- 敏感目录（密钥、`.env`、证书）在报告中可只写路径与「存在/缺失」，不打印内容。

## 常用命令提示（在 bash 中按环境选用）

```bash
# 文件数
find "$SRC" -type f | wc -l
find "$DST" -type f | wc -l

# 相对路径列表排序后 diff
(cd "$SRC" && find . -type f | sort) > /tmp/src.list
(cd "$DST" && find . -type f | sort) > /tmp/dst.list
diff -u /tmp/src.list /tmp/dst.list

# 单文件校验
shasum -a 256 "$SRC/path" "$DST/path"
```
